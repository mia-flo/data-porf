Database design with SQLAlchemy and MySQL

Objective: Normalize a database to reduce redundancy and improve query efficiency

Results: Turned the database into 6 tables normalized to the fourth normal form (4NF) and created audit tables to track changes

Languages: Python, SQL

Tools:

 Jupyter notebooks:

  • SQLAlchemy
  • Pandas

 MySQL Workbench

Methods:

Python:

  • UUID4 in to generate unique user ids for 92 customers
  • Merging dataframes with Pandas
  • Lambda function and apply() to calculate total sales over 2,823 rows
MySQL:
  • Recursive common table expressions
  • Window functions to sum total sales over individual orders
  • Stored procedures to summarize the sales for the most recent year and most recent month
  • Database triggers to track updates in product MSRP and customer address

Data set: sourced from kaggle

  • 2,823 records

Database normalization to 4NF

The large, 25 column table was broken down into smaller, more manageable tables.

To improve query efficiency and reduce redundancy:

  • created unique customer and address ids
  • dropped the deal-size column
  • used SQLAlchemy in Jupyter notebooks to organize the data into 6 tables
