Customer segmentation analysis based on purchase history using SQL

Objective: To group customers based on their purchasing behavior

Languages: SQL, Python

Tools:

-MS SQL Server

-Jupyter notebooks: Scipy, statsmodels, pandas, numpy, math, matplotlib, seaborn

Methods:

-SQL: data cleaning, common table expressions, running total calculations with window functions

-Python: probability density functions for probability calculations

Results:

Data set: sourced from kaggle - 541,909 records aggregated with SQL into 18,528 orders, +8,900 returns, and +134,000 inventory records for 4,343 unique customers across 36 countries.

Customer groups

Regular: 3 or more months with consecutive orders

Return: 3 or more orders in non-consecutive months

New: 2 orders

One-time: 1 order

Percent contribution of each customer type to key metrics

Regular customers, by far, contributed the most to the business in the key metrics of gross, number of orders, and number of units sold despite making up only 20% of the customer base.
